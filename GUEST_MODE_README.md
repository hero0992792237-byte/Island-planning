## 🎁 为什么选择登录？

虽然访客模式已经足够好用，但**登录后您可以获得更多福利**：

### 🎁 邀请奖励
- 每成功邀请 1 位好友
- 双方各获得 **50 次免费 API 调用**
- 邀请越多，额度越高

### ☁️ 云端同步
- 偏好设置保存到云端
- 多设备数据自动同步
- 清除浏览器也不会丢失

### 👥 好友系统
- 添加好友一起规划行程
- 查看好友的旅行计划
- 分享推荐目的地

### 📊 更多额度
- 通过邀请和使用持续获得额度
- 不再受访客模式限制

## 💻 技术实现

### 修改的文件

1. **src/context/AppContext.tsx** (第 99-108 行)
   - 将 `isLoggedIn` 默认值改为 `true`
   - 创建默认访客用户
   - 设置访客额度为 1000 次

2. **src/components/ExpandedView.tsx** (第 191-235 行)
   - 调整内容显示逻辑
   - 确保访客模式下正常显示主界面

## ⚠️ 注意事项

### 访客模式限制

1. **数据持久化**: 偏好设置保存在 `localStorage`，清除浏览器数据会丢失
2. **API 额度**: 1000 次免费额度用完需要登录或等待重置
3. **社交功能**: 邀请、好友等功能需要登录

### 推荐使用流程

```
打开应用 → 访客使用 → 体验功能 → 点击登录 → 获得更多额度 → 使用云端同步
```

## 🔧 如何切换回登录模式？

如果您希望恢复严格的登录验证：

1. 打开 `src/context/AppContext.tsx`
2. 将第 100 行的 `isLoggedIn: true` 改回 `isLoggedIn: false`
3. 删除或注释掉第 101-108 行的默认用户配置
4. 重启开发服务器

```typescript
userAuth: {
  isLoggedIn: false,  // 恢复登录验证
  user: null,
  token: null,
  friends: [],
  inviteCode: '',
  inviteCount: 0,
  freeApiQuota: 0,
},
```
